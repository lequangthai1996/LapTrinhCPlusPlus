nguyenchiemminhvu

kiểu int32_t, int16_t cũng đều là kiểu số nguyên cả, nó lưu giá trị hoàn toàn giống với kiểu int thông thường. Nhưng khi sử dụng kiểu int thì tùy thuộc vào compiler đang sử dụng mà nó có kích thước khác nhau. Có khi kiểu int có kích thước 16 bits = 2 bytes, trong visual studio 2015 thì kiểu int quy định là 32 bits = 4 bytes.

Sử dụng thư viện cstdint chỉ là để biết rõ kích thước vùng nhớ đang sử dụng thôi bạn. Để sau này khi cần code cho ứng dụng không cần lưu trữ giá trị lớn, thì có thể sử dụng các kiểu int16_t hay int8_t sẽ tiết kiệm bộ nhớ hơn kiểu int32_t hay int.

Hoặc khi cần kiểu int có kích thước lớn, thay vì khai báo long long int thì chỉ cần khai báo int64_t là được.

Code::Blocks vẫn có chuẩn C++11 và mới nhất là C++14 nha bạn, chỉ là nó mặc định xài C++98 thôi.

Để cho compiler biết mình viết code theo chuẩn nào, bạn vào Settings ⇒ Compiler... Nó hiện ra cửa sổ có 1 cái bảng. Bạn đánh dấu tick vào ô có nội dung bên trái là Have g++ follow the C++14 ISO C++ language standard để theo chuẩn C++14 hoặc Have g++ follow the C++11 ISO C++ language standard để theo chuẩn C++11.
